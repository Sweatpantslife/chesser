# engines/

Chess engine binaries and neural-network weights live here. **Nothing in this
folder is committed** (except this file) — it is all generated by:

```bash
pnpm setup:engines
```

After setup you'll have:

```
engines/
├── bin/
│   ├── stockfish        # official Stockfish release binary
│   └── lc0              # Lc0, built from source (runs the Maia nets)
├── networks/
│   ├── maia-1100.pb.gz  # human-like nets, one per rating band
│   ├── maia-1500.pb.gz
│   └── maia-1900.pb.gz
└── manifest.json        # what got installed; the server reads this at boot
```

The backend validates every path in `manifest.json` against the filesystem, so
a partial install (e.g. Stockfish only) still yields a working app — the UI just
offers fewer opponents.

## Notes

- **Stockfish** ships official Linux binaries; setup downloads the fastest one
  your CPU supports (avx512 / bmi2 / avx2 / …).
- **Lc0** has no official Linux binary, so it is built from source with the
  CPU/Eigen backend. This needs `meson` (installed via pip if absent), `ninja`,
  a C++ compiler, and Eigen headers (`libeigen3-dev`).
- **Maia** weights come from the [CSSLab/maia-chess](https://github.com/CSSLab/maia-chess)
  project. Maia is meant to be run at one node (`go nodes 1`) — it then plays the
  move a human of that rating would most likely choose.

Override behaviour with env vars, e.g. `SKIP_LC0=1 pnpm setup:engines` for a
Stockfish-only install, or `ONLY=stockfish pnpm setup:engines`.
